# 바이브 코딩 데모 튜토리얼

이 문서는 vibeCodingTest 라이브 데모에서 실제로 사용한 프롬프트를 시간순으로 기록한 재현용 가이드다. 각 단계는 "무엇을 만들 것인가 → 어떤 프롬프트로 시켰는가 → 산출물" 흐름으로 구성된다.

전체 흐름은 다음 4단계로 진행됐다.

1. **PRD 초안 다지기** — 아이디어를 prod-prd.md로 정리
2. **PRD 확정** — 기술스택 제안 → docs/prd.md 생성
3. **GitHub 공개 + CLAUDE.md** — 원격 저장소 업로드 + 프로젝트 가이드 작성
4. **실행 계획 + GitHub Issue 생성** — 1시간 데모용 태스크 분할 + Issue 등록

요구사항을 한 문장에 모두 담는다: 작업내용 / 작업목적 / 인수조건(AC) / 의존성.

데모 머신에 `gh`가 없으면 Claude가 설치 명령을 안내한다. 사용자는 그 명령을 그대로 입력해 진행한다.

> winget install --id GitHub.cli

> gh auth login

> ok

→ 산출물: vibeCodingTest 레포의 GitHub Issues에 태스크 단위로 등록 (작업내용/목적/AC/의존성 포함)

## 세션 5 (2026-04-25 늦은 밤) — 튜토리얼 자체 자동화

데모의 마지막 패턴: 데모를 진행하면서 그 데모 자체를 문서로 남기는 슬래시 명령을 호출한다.

```
/tutorial
```

`.claude/commands/tutorial.md`에 정의된 명령으로, `~/.claude/projects/<this project>/`의 프롬프트 히스토리를 읽어 이 파일을 갱신한다.

## 데모 진행자를 위한 메모

- **언어 일관성**: 사용자 발화·커밋 메시지·문서는 한국어로, 코드와 식별자는 영어로 유지한다.
- **결정은 PRD에 가둔다**: [docs/prd.md](docs/prd.md)가 단일 진실 공급원. 데모 중 즉흥 결정으로 흔들지 않는다.
- **추천 → 동의** 2단 패턴: "1번/2번 중 추천?" → "1번으로 진행해줘"는 청중에게 의사결정 흐름을 보여주는 좋은 장치다.
- **세션이 끊기면**: 새 세션 첫 메시지로 "지금까지 한 일과 앞으로 할 일을 알려줘"를 치면 바로 컨텍스트가 복원된다.
- **계획 파일은 레포 안으로**: `~/.claude/plans/`에 생긴 산출물은 `docs/`로 옮겨 git 추적에 포함시킨다.
